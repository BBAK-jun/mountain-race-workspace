# 등산복 입고 뛰어 — 온라인 멀티플레이어 & 숨겨진 효과 PRD

> "각자 기기에서 접속해 캐릭터를 등록하고, 정체를 알 수 없는 비밀 효과를 쥔 채 레이스를 관전하는 온라인 파티 게임"

기준 문서:

- `docs/mountain-race-product-prd.md` (제품 PRD)
- `docs/mountain-race-technical-prd.md` (기술 PRD)
- `docs/mountain-race-mvp-guide.md` (MVP 가이드)

## 1. 개요

| 항목       | 내용                                                                                              |
| ---------- | ------------------------------------------------------------------------------------------------- |
| 기능명     | 온라인 멀티플레이어 + 숨겨진 랜덤 효과                                                           |
| 한 줄 요약 | 각자 기기에서 방에 접속해 캐릭터를 등록하고, 정체 모를 비밀 효과를 발동할지 말지 선택하는 관전 게임 |

## 2. 배경

### 2.1 현재 상태

- 게임은 하나의 브라우저에서 2~8명 캐릭터를 등록하고 자동 레이스를 관전하는 로컬 파티 모드
- 백엔드(`apps/api`)는 Hono 스타터로 health 엔드포인트만 존재
- 이벤트 시스템(skill, ultimate, global, target)과 대사 시스템은 클라이언트에서 완전 구현됨

### 2.2 한계

- 같은 기기에 모여야 플레이 가능 → 원격 모임/온라인에서 사용 불가
- 관전만 가능하고 플레이어가 레이스에 개입할 수단이 없음 → 관전 피로
- 다른 사람 화면에서 내 캐릭터가 어떻게 보이는지 공유 불가 → 바이럴 약화

### 2.3 목적

- 각자 기기에서 접속하는 온라인 멀티플레이로 원격 파티 지원
- "숨겨진 효과" 메카닉으로 관전 중 전략적 개입 요소 추가
- 서버 권한 모델로 게임 상태 동기화 + 숨겨진 정보 보호

## 3. 핵심 원칙

| 원칙              | 설명                                                              |
| ----------------- | ----------------------------------------------------------------- |
| 숨겨진 정보 = 긴장 | 내가 가진 효과가 부스트인지 자해인지 모르는 상태가 핵심 재미       |
| 선택 = 전략       | 발동할지 말지는 전적으로 플레이어의 선택. 안 쓸 수도 있음          |
| 관전 + 참여       | 기본은 관전이지만, 한 번의 발동 기회가 게임을 뒤집을 수 있음       |
| 아쉬움 = 리플레이  | 결과 화면에서 미사용 효과가 공개되면 "그거 부스터였는데!" 유발      |
| 서버가 진실       | 게임 시뮬레이션과 효과 배정은 서버에서만 처리, 클라이언트는 렌더만  |

## 4. 사용자 흐름

```text
[랜딩] → [방 만들기 / 참가] → [로비] → [레이스 관전 + 효과 발동] → [결과]
              ↑                                                        |
              └──────────────────── 다시 하기 ←────────────────────────┘
```

### 4.1 랜딩페이지

- 기존 3D 배경 유지
- CTA 변경: "게임 시작" → "방 만들기" / "방 참가"
- "방 만들기": 서버에 방 생성 요청 → 4자리 코드 발급 → `/lobby` 이동
- "방 참가": 코드 입력 필드 표시 → 유효한 코드 입력 시 → `/lobby` 이동

### 4.2 로비 (`/lobby`)

1. 상단에 방 코드 대형 표시 (복사 버튼)
2. 접속한 플레이어 목록 실시간 표시 (WebSocket)
3. 각자 자기 캐릭터만 설정: 닉네임, 얼굴 업로드, 컬러 선택
4. "준비" 토글 버튼
5. 전원 준비 완료 시 호스트에게 "등산 시작" 버튼 활성화
6. 호스트가 시작 → 서버가 비밀 효과 배정 → 카운트다운 → `/race`로 전환

### 4.3 레이스 관전 + 효과 발동 (`/race`)

1. 기존 레이스 관전 경험 유지 (3D 산악 코스, 자동 레이스, 이벤트, 대사)
2. 게임 상태는 서버에서 수신 (로컬 tick 없음)
3. 내 화면 HUD 하단에 **"? 발동" 버튼** 표시
   - 펄스 애니메이션으로 존재감 부여
   - 효과가 뭔지 모르는 상태에서 누를지 판단
4. 버튼 클릭 시:
   - 서버에 `ActivateEffect` 전송
   - 서버가 효과 적용 + `EffectReveal` 전체 브로드캐스트
   - 모든 클라이언트에서 카메라 줌 + 공개 연출
5. 안 누르면: 효과 미사용, 레이스 종료까지 버튼 유지
6. 다른 플레이어 캐릭터 위에 "?" 배지 표시 (미발동 상태)

### 4.4 결과 화면 (`/result`)

- 기존 순위 + MVP 시상 유지
- 추가: **숨겨진 효과 공개 카드**
  - 각 플레이어가 어떤 효과를 받았는지 전체 공개
  - 사용 여부 표시 ("발동!" / "미사용")
  - 미사용 효과가 좋은 효과였으면 "아깝다!" 라벨

## 5. 기능 명세

### 5.1 방 시스템

| 항목       | 상세                                                        |
| ---------- | ----------------------------------------------------------- |
| 방 코드    | 4자리 영문 대문자 (예: "ABCD")                              |
| 최대 인원  | 8명 (기존 MAX_PLAYERS 유지)                                 |
| 최소 인원  | 2명 (기존 MIN_PLAYERS 유지)                                 |
| 호스트     | 방을 만든 사람. 시작 권한 보유                              |
| 호스트 이탈| 다음 접속자에게 호스트 이전                                  |
| 방 수명    | 결과 화면 후 5분 무활동 시 자동 삭제                        |
| 재접속     | 같은 방 코드 + playerId로 재접속 가능 (레이스 중에도)       |

### 5.2 로비

| 항목       | 상세                                                        |
| ---------- | ----------------------------------------------------------- |
| 캐릭터 설정| 닉네임, 얼굴 이미지(Data URL), 컬러(서버 자동 배정)         |
| 준비 상태  | 토글. 전원 준비 시 호스트에게 시작 버튼 활성화               |
| 실시간 동기| 플레이어 참가/이탈/준비 변경 즉시 반영 (WebSocket)           |
| 시작 조건  | 2명 이상 + 전원 준비 + 호스트가 시작 클릭                   |

### 5.3 숨겨진 효과 시스템

이 게임의 핵심 차별화 메카닉.

#### 효과 풀

**좋은 효과 (약 40%)**

| 효과       | 설명                              | 적용 대상 |
| ---------- | --------------------------------- | --------- |
| booster    | 2초간 속도 2.5배                  | 자기 캐릭터 |
| wind_ride  | 1.5초간 속도 1.8배                | 자기 캐릭터 |
| shield     | 다음 방해 효과 1회 완전 무효화    | 자기 캐릭터 |

**나쁜 효과 (약 40%)**

| 효과         | 설명                              | 적용 대상 |
| ------------ | --------------------------------- | --------- |
| self_trip    | 1.2초 스턴 (스스로 넘어짐)        | 자기 캐릭터 |
| ankle_weight | 3초간 속도 0.7배 감속             | 자기 캐릭터 |
| magnet       | 2등과 진행도 차이의 50% 만큼 후퇴 | 자기 캐릭터 |

**와일드카드 (약 20%)**

| 효과          | 설명                                      | 적용 대상 |
| ------------- | ----------------------------------------- | --------- |
| mystery_swap  | 가장 가까운 상대와 진행도 교환            | 자기 + 대상 |
| earthquake    | 전원 0.8초 스턴 (본인 포함)               | 전체 |

#### 배정 규칙

- 레이스 시작 직전, 서버가 각 플레이어에게 효과 풀에서 1개를 랜덤 배정
- 효과 종류와 내용은 서버 메모리에만 저장
- 클라이언트에는 `hasHiddenEffect: true`만 전달 (내용 미공개)
- 배정 확률은 균등 랜덤 (순위 무관, 레이스 시작 전이므로)

#### 발동 규칙

- 플레이어가 레이스 중 "? 발동" 버튼을 클릭해야만 발동
- 레이스당 1회만 가능 (발동 후 버튼 사라짐)
- 발동하지 않아도 패널티 없음
- 레이스 종료 후에는 발동 불가
- 이미 완주한 캐릭터의 효과 발동: 완주 후에도 발동 가능하지만, 이동 관련 효과는 무의미

#### 발동 시 흐름

1. 클라이언트 → 서버: `ActivateEffect` 메시지
2. 서버: 효과 적용 (캐릭터 상태 변경)
3. 서버 → 전체 클라이언트: `EffectReveal` 브로드캐스트
4. 클라이언트 연출:
   - 카메라를 해당 캐릭터로 줌
   - "?" 아이콘 → 실제 효과 아이콘으로 전환 애니메이션
   - 이벤트 로그에 공개 메시지 추가
   - 2초간 화면 중앙 대형 알림

#### 결과 화면 공개

- 레이스 종료 시 서버가 모든 플레이어의 효과 정보를 `RaceResult`에 포함
- 사용 여부, 효과 종류, 좋은/나쁜/와일드카드 분류 표시
- 미사용 좋은 효과에 "아깝다!" 라벨, 미사용 나쁜 효과에 "현명한 선택!" 라벨

### 5.4 레이스 동기화

| 항목            | 상세                                                    |
| --------------- | ------------------------------------------------------- |
| 권한 모델       | 서버 권한 (Server-Authoritative)                        |
| 시뮬레이션 위치 | Durable Object 내부 (`alarm()` API)                     |
| tick 간격       | 16ms (alarm)                                            |
| 브로드캐스트    | 50ms 간격으로 상태 스냅샷 전송                          |
| 클라이언트 역할 | 수신한 상태를 렌더링만 (보간 적용)                      |
| 기존 이벤트     | skill/ultimate/global/target 모두 서버에서 처리          |
| 종료 조건       | 기존과 동일 (전원 완주 또는 선두 완주 후 유예 시간)     |

### 5.5 결과 화면 확장

기존 결과 화면에 추가:

- 숨겨진 효과 공개 카드 섹션
- 효과 발동 타이밍 표시 (레이스 경과 시간 기준)
- "이 판의 숨겨진 효과" 요약 테이블

## 6. 기술 아키텍처

### 6.1 인프라

```text
┌─────────────────┐     WebSocket      ┌──────────────────────────┐
│  Player Browser  │ ◄──────────────► │  Cloudflare Worker       │
│  (apps/web)      │                   │  ├─ Hono Router          │
│  - React + R3F   │     REST (RPC)    │  ├─ /rooms REST routes   │
│  - hc<AppType>   │ ◄──────────────► │  └─ RaceRoom DO          │
│  - Zustand       │                   │     ├─ WebSocket handler │
└─────────────────┘                   │     ├─ Game simulation   │
                                       │     └─ Hidden effects    │
                                       └──────────────────────────┘
```

### 6.2 공유 패키지

| 패키지                    | 경로               | 내용                                        |
| ------------------------- | ------------------ | ------------------------------------------- |
| `@mountain-race/types`    | `packages/types`   | Character, GameEvent, WS 메시지, Room, 효과 |
| `@mountain-race/game-logic`| `packages/game-logic`| EventSystem, DialogueSystem, balance, data |

### 6.3 Hono RPC

```ts
// apps/api/src/index.ts
import { Hono } from "hono";
import { roomRoutes } from "./routes/room";

const app = new Hono();
const routes = app.route("/rooms", roomRoutes);

export default app;
export type AppType = typeof routes;
```

```ts
// apps/web — 타입 세이프 클라이언트
import { hc } from "hono/client";
import type { AppType } from "@mountain-race/api"; // tsconfig project ref

const client = hc<AppType>(import.meta.env.PUBLIC_API_URL);

// REST: 방 생성
const res = await client.rooms.$post({ json: { hostName: "산악인" } });

// WebSocket: RPC 모드
const ws = client.rooms[":code"].ws.$ws({ param: { code: "ABCD" } });
```

## 7. WebSocket 메시지 프로토콜

### 7.1 Client → Server

| 메시지           | 페이로드                                             | 설명                     |
| ---------------- | ---------------------------------------------------- | ------------------------ |
| `SetCharacter`   | `{ name, faceImage, color }`                         | 캐릭터 정보 설정         |
| `SetReady`       | `{ ready: boolean }`                                 | 준비 상태 토글           |
| `StartRace`      | `{}`                                                 | 호스트만. 레이스 시작    |
| `ActivateEffect` | `{}`                                                 | 비밀 효과 발동 요청      |

### 7.2 Server → Client

| 메시지            | 페이로드                                            | 설명                     |
| ----------------- | --------------------------------------------------- | ------------------------ |
| `RoomState`       | `{ players[], phase, hostId, roomCode }`            | 전체 방 상태 스냅샷      |
| `PlayerJoined`    | `{ player }`                                        | 새 플레이어 참가         |
| `PlayerLeft`      | `{ playerId }`                                      | 플레이어 이탈            |
| `PlayerUpdated`   | `{ playerId, changes }`                             | 캐릭터/준비 상태 변경    |
| `Countdown`       | `{ seconds }`                                       | 레이스 카운트다운        |
| `HasHiddenEffect` | `{ hasEffect: true }`                               | 비밀 효과 보유 알림      |
| `GameTick`        | `{ characters[], rankings[], events, elapsed, ... }` | 게임 상태 스냅샷 (50ms)  |
| `EffectReveal`    | `{ playerId, effectType, category, description }`   | 효과 공개 연출           |
| `RaceResult`      | `{ rankings[], stats[], hiddenEffects[] }`          | 최종 결과 + 효과 전체 공개|

## 8. UI/UX 상세

### 8.1 "? 발동" 버튼

- 위치: HUD 하단 중앙
- 크기: 최소 48x48px 터치 영역 (모바일 우선)
- 디자인: 둥근 원형, "?" 텍스트, 반투명 배경
- 애니메이션: 부드러운 펄스 (scale 1.0 ↔ 1.05, 2초 주기)
- 클릭 시: 즉시 사라짐 + 서버에 요청
- 비활성: 이미 사용 또는 레이스 종료 시

### 8.2 "?" 배지

- 다른 플레이어 캐릭터 머리 위에 표시
- 작은 원형 배지, "?" 텍스트
- 해당 플레이어가 효과 발동 시 → 실제 효과 아이콘으로 전환
- 레이스 종료까지 미발동이면 "?" 유지

### 8.3 효과 공개 연출

1. 카메라: 해당 캐릭터로 `event_zoom` 전환 (1초)
2. "?" → 효과 아이콘 전환 (스프링 애니메이션)
3. 화면 중앙 대형 알림: "🎲 {name}의 비밀 효과: {effectName}!"
4. 이벤트 로그에 메시지 추가
5. 좋은 효과면 초록 테두리, 나쁜 효과면 빨간 테두리, 와일드카드면 보라 테두리

### 8.4 결과 화면 효과 카드

- 각 플레이어별 카드 형태
- 효과 아이콘 + 이름 + 설명
- 사용 여부 배지 ("발동!" / "미사용")
- 미사용 좋은 효과: "아깝다!" 라벨 (황금색)
- 미사용 나쁜 효과: "현명한 선택!" 라벨 (녹색)
- 미사용 와일드카드: "모험을 두려워하다니..." 라벨 (보라색)

## 10. 로컬 모드 호환

온라인 멀티플레이를 추가하되, 기존 로컬 파티 모드도 유지한다.

| 항목         | 온라인 모드                 | 로컬 모드                    |
| ------------ | --------------------------- | ---------------------------- |
| 진입         | 방 만들기/참가 → 로비       | 기존 설정 화면               |
| 캐릭터 설정  | 각자 기기에서 1명           | 한 기기에서 2~8명            |
| 시뮬레이션   | 서버 (Durable Object)       | 클라이언트 (기존 tick)       |
| 숨겨진 효과  | 서버 배정 + 발동 버튼       | 없음 (기존 이벤트 시스템)    |
| WebSocket    | 필수                        | 불필요                       |
| 라우트       | / → /lobby → /race → /result| / → /setup → /race → /result|

랜딩에서 "로컬 플레이"를 선택하면 기존 `/setup` 플로우를 그대로 탄다.
